DMP Section 3: Data Format and Metadata Standards Employed

Open File Formats

This project aims to achieve open access to the fullest extent by releasing the data to the public.

This project will generate data in Microsoft formats, but will create open-access CSVs and PDFs for public use.

As ESRI is the dominate format for all GIS projects, they are accepted as a preservation format by the Library of Congress. Some ESRI file formats can be converted to CSV or TIFF, and where possible we will. However, as different layers of information can get lost during conversion, we intend to include the original ESRI file alongside any conversions in the final dataset. For public access, we recommend the use of QGIS https://www.qgis.org/en/site/, an open-source software that can open ESRI files.

DCAT-US Metadata Schema

The final data package will include a DCAT-US version 1.1 (https://resources.data.gov/resources/dcat-us/) .JSON metadata file, which is the federal standard for datasets and required for compliance with the USDOT Public Access Plan.

DMP Section 4: Access Policies

Using Tribal or Indigenous Peoples Data

As this project will use data from Indigenous Peoples, this project will strictly adhere to the CARE Principles to ensure that the data used for this project is grounded in Indigenous worldviews and uses Indigenous data responsibly and ethically.
